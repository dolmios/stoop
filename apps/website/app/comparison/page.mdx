# Stitches vs Stoop

<Icon name="FileText" size={24} style={{ display: "inline", verticalAlign: "middle", marginRight: "8px" }} /> Technical comparison between Stitches and Stoop CSS-in-JS libraries

## Overview

Stoop is a minimalist implementation of Stitches' high-level features. It provides a similar API for `styled`, `css`, and variants, but with some key differences.

## Key Differences

### Theme System

**Stitches:** Theme tokens resolve to runtime values injected via JavaScript.

**Stoop:** Theme tokens resolve to CSS variables (`var(--colors-primary)`), enabling instant theme switching without recompiling CSS.

### Multi-Theme Support

**Stitches:** Requires manual theme management.

**Stoop:** Built-in multi-theme support via `Provider` component with automatic CSS variable switching.

### Build-Time CSS Extraction

**Stitches:** Supports build-time CSS extraction for zero-runtime CSS.

**Stoop:** Runtime-only CSS generation. If you need build-time extraction, consider [Vanilla Extract](https://vanilla-extract.style) or [Panda CSS](https://panda-css.com).

### Compound Variants

**Stitches:** Supports compound variants (combining multiple variant conditions).

**Stoop:** Does not currently support compound variants (planned for future release). Variants work independently, which is sufficient for most use cases. You can achieve similar results using conditional `css` props or wrapper components.

### Utility Functions

**Stitches:** Advanced utility function system with extensive built-in utilities.

**Stoop:** Flexible utility function system with the same capabilities. You can create any custom utilities you need. Stitches provides more built-in utilities out of the box, but Stoop's utility system is equally powerful and flexible.

## API Differences

### Variants API

**Stitches:**
```typescript
const Button = styled("button", {
  padding: "$medium",
  variants: {
    size: {
      small: { padding: "$small" },
      large: { padding: "$large" },
    },
  },
});
```

**Stoop:**
```typescript
const Button = styled("button", {
  padding: "$medium",
}, {
  size: {
    small: { padding: "$small" },
    large: { padding: "$large" },
  },
});
```

**Key Difference:** In Stitches, variants are a property in the second parameter. In Stoop, variants are the third parameter.

### Theme Configuration

**Stitches:**
```typescript
const { styled } = createStitches({
  theme: { ... },
});
```

**Stoop:**
```typescript
const { styled, Provider, useTheme } = createStoop({
  theme: lightTheme,
  themes: {
    light: lightTheme,
    dark: darkTheme,
  },
});
```

**Key Difference:** Stoop adds `themes` object for multi-theme support and automatically provides `Provider` and `useTheme`.

## Migration Considerations

When migrating from Stitches to Stoop:

1. **Theme tokens:** Work the same way (`$medium`, `$primary`, etc.)
2. **Component API:** Mostly identical (variants move to third parameter)
3. **CSS variables:** Stoop uses CSS variables instead of runtime injection (better performance)
4. **Multi-theme:** Requires `Provider` wrapper in Stoop (but much easier to use)
5. **Compound variants:** Need to be refactored to use conditional `css` props (most projects don't use compound variants)
6. **Utilities:** Custom utilities work the same way - you may need to recreate Stitches' built-in utilities if you use them
7. **getCssText():** Simpler in Stoop - no theme parameter needed, always includes all themes

See the [Migration Guide](/migration) for detailed migration steps.

## When to Use Stoop

- You want a Stitches-like API with CSS variable-based theming (instant theme switching)
- You need built-in multi-theme support with Provider component
- You're using Next.js App Router or React 18 Server Components
- You want an actively maintained library (Stitches is no longer maintained)
- You prefer a simpler API (no theme parameter needed for `getCssText()`)
- You don't need compound variants (most projects don't)

## When to Use Stitches

- You need compound variants (Stoop planned for future)
- You want build-time CSS extraction
- You prefer Stitches' built-in utility functions (though Stoop utilities are equally capable)
- You have an existing Stitches codebase and don't want to migrate

## Development Status

**Stitches:** No longer actively maintained (as of 2024).

**Stoop:** Currently in active development (beta).
